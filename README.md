# retail-demand-forecast

## 📋 Описание проекта
Этот проект представляет собой систему управления складскими запасами на основе прогнозирования спроса с использованием временных рядов.

Цель проекта — минимизировать риски дефицита товара (out-of-stock) и оптимизировать затраты на хранение. Система анализирует исторические данные о продажах, строит прогноз на будущее и рассчитывает рекомендуемый объем закупки с учетом страхового запаса.

## Технологический стек
- **Python** (Pandas, Numpy)
- **Prophet** — для прогнозирования временных рядов.
- **Scikit-learn** — для оценки метрик (MAE, RMSE).
- **Streamlit** — для создания интерактивного дашборда.
- **Plotly** — для визуализации данных.

## Этапы реализации
1. **Data Collection**: Загрузка и объединение данных Online Retail II из UCI Machine Learning Repository (периоды 2009-2011).
2. **Data Cleaning**: Обработка пропусков, удаление отмен (Invoice 'C') и фильтрация выбросов.
3. **EDA**: Анализ недельной и сезонной цикличности продаж, выявление аномалий.
4. **Modeling**: Сравнение Baseline (Moving Average) и модели Prophet для учета сезонности.
5. **Business Logic**: Автоматический расчет страхового запаса (Safety Stock) и точки заказа.
6. **Deployment**: Создание интерфейса на Streamlit для операционного управления.

## Структура файлов
- `app.py`: Код Streamlit-приложения.
- `model.pkl`: Сохраненная обученная модель.
- `requirements.txt`: Список необходимых библиотек.
- `inventory_optimization.ipynb`: Основной блокнот с исследованием данных.
